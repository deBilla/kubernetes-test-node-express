# CLAUDE.md

## Development Approach

- Always follow Test-Driven Development (TDD): write failing tests first, then implement the minimum code to pass, then refactor.
- Follow best software engineering practices with clean code: meaningful naming, single responsibility, DRY, KISS, and SOLID principles.

## Git Workflow

- Always ask for explicit permission before committing or pushing changes. Never commit or push automatically.
- Before committing, check all changes for PII (passwords, API keys, tokens, emails, credentials, personal data). Flag any findings before proceeding.
- Before committing, update the README.md to reflect any changes made.
- Before committing, update the CHANGELOG.md with a summary of changes. Keep the changelog as a separate file from the README.

## Domain Expertise

- This project involves Kubernetes and Helm. Apply expert-level knowledge when working with deployments, services, ingress, ConfigMaps, Secrets, Helm charts, values files, and templating.
